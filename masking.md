# 마스킹 컬럼 권한 신청 기능 요구사항

## 1. 개요

SQL 쿼리 결과에서 마스킹된 컬럼에 대해 사용자가 권한을 신청하고,
신청 상태를 조회 및 관리할 수 있는 기능을 제공한다.

본 기능은 다음 두 가지 진입 경로를 지원한다:

1. 쿼리 실행 결과에서 바로 신청
2. 별도의 권한 관리 페이지에서 신청

---

## 2. 핵심 개념 정의

### 2.1 마스킹 컬럼

- SQL 쿼리 결과에서 값이 마스킹 처리된 컬럼
- 예: `***`, `partial masking`

### 2.2 권한 상태

| 상태     | 설명      |
| -------- | --------- |
| NONE     | 권한 없음 |
| PENDING  | 승인 대기 |
| APPROVED | 승인됨    |
| REJECTED | 거절됨    |

### 2.3 권한 단위

- 기본 단위: **Column Level**
- 하나의 요청에서 여러 컬럼을 동시에 신청할 수 있음

---

## 3. 기능 요구사항

## 3.1 SQL 쿼리 결과

### 3.1.1 마스킹 표시

- 마스킹된 컬럼은 UI에서 구분되어야 함
  - 아이콘 또는 스타일 차별화

### 3.1.2 권한 상태 표시

- 각 컬럼별 권한 상태를 표시
  - 신청 중인 경우: PENDING 표시
  - 승인된 경우: 일반 데이터로 표시

---

## 3.2 쿼리 결과에서 권한 신청

### 3.2.1 진입 방식

- 마스킹 컬럼 클릭 시 권한 신청 UI 진입

### 3.2.2 자동 설정 값

쿼리 컨텍스트 기반으로 다음 값 자동 설정:

- 아래 설정을 페이지에서 기억하기 위해 query 로 처리 (페이지 스텝 상태도 쿼리로 처리)
  - schema
  - table
  - column

### 3.2.3 신청 UI

- 테이블 권한 신청과 동일한 ui 인 페이지로 구성성
- Step 기반 UI로 구성 (쿼리 기반 상태 처리리)

---

## 3.3 권한 신청 프로세스 (Step Flow)

### Step 1. 대상 확인

- schema (기본값 설정)
- table (기본값 설정)
- column (기본값 설정)
- 필요 시 수정 가능

---

### Step 2. 마스킹 컬럼 선택

- 마스킹된 컬럼 목록 제공
- 다중 선택 가능

---

### Step 3. 사유 입력

- 권한 신청 사유 입력 필드 제공
- 최소 입력 길이 제한 가능

---

### Step 4. 검토 및 제출

- 선택한 컬럼 목록 요약
- 입력한 사유 확인
- 최종 제출

## 참고로, 스텝 3과, 4를 합칠 수 있다면 합쳐도 좋을듯!

## 3.4 마스킹 권한 관리 페이지

### 3.4.1 목록 조회

다음 정보를 포함한 리스트 제공:

- schema
- table
- column(s)
- 상태 (PENDING / APPROVED / REJECTED)
- 신청 사유
- 신청일
- 처리일 (선택)

---

### 3.4.2 "신청하기" 기능

- 별도의 "신청하기" 버튼 제공
- 클릭 시 Step 기반 신청 프로세스 시작

---

## 3.5 권한 페이지에서 신청 (수동 신청)

### 3.5.1 Step Flow

#### Step 1. Schema / Table 선택

- schema 선택
- table 선택 (schema 의존)

#### Step 2. 마스킹 컬럼 선택

- 선택한 테이블의 마스킹 컬럼 목록 제공
- 다중 선택 가능

#### Step 3. 사유 입력

#### Step 4. 검토 및 제출

---

## 3.6 쿼리 기반 신청 vs 수동 신청

| 항목         | 쿼리 기반 신청 | 수동 신청      |
| ------------ | -------------- | -------------- |
| schema/table | 자동 설정      | 사용자 선택    |
| column       | 자동 선택      | 사용자 선택    |
| UX           | 빠른 신청      | 탐색 기반 신청 |

---

## 4. 추가 요구사항

### 4.1 중복 신청 방지

- 이미 PENDING 상태인 컬럼은 재신청 제한 또는 안내 표시

---

### 4.2 상태 실시간 반영

- 권한 승인 시, 이후 쿼리에서 마스킹 해제된 데이터 제공

---

### 4.3 다중 컬럼 신청

- 여러 컬럼을 하나의 요청으로 묶어서 신청 가능

---

### 4.4 사용자 피드백

- 신청 완료 시 성공 메시지 제공
- 상태 변경 시 사용자에게 인지 가능하도록 표시

---

### 4.5 감사 및 추적 (옵션)

- 누가 어떤 컬럼을 언제 신청했는지 기록

---

## 5. UX 설계 원칙

- 쿼리 결과에서 **최소 클릭으로 신청 가능해야 함**
- 자동 설정을 최대한 활용하여 입력 최소화
- 상태를 명확하게 시각적으로 표현
- 동일한 신청 프로세스를 재사용 (쿼리 / 권한 페이지)
