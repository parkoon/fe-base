## 1. 개요

### 1.1 문제 정의

현업 사용자가 정보계 DB에 직접 접근하고 있어 데이터 유출·보안 통제가 불가하며,
접근 권한이 수작업으로 관리되어 퇴사/부서이동 시 권한 회수가 누락되는 구조적 문제가 있다.

금융권 감사 대응 차원에서도 SQL 실행 이력, 권한 신청/승인 이력, 개인정보 조회 이력을
위변조 불가한 형태로 1년 이상 보관해야 하나, 현재 체계에서는 이를 충족하지 못한다.

### 1.2 제안 솔루션

웹 기반 SQL 에디터(웹쿼리)와 권한관리 어플리케이션을 통합 구축하여,
사용자는 반드시 웹을 통해서만 정보계 DB에 접근하도록 강제한다.

모든 접근은 **권한 신청 → 다단계 결재 승인 → 자동 권한 부여/회수** 플로우를 거치며,
쿼리 실행 시 개인정보 마스킹·건수 제한·조회이력 적재가 자동 적용된다.

## 3. 유저 스토리 및 완료 기준

### 3.1 권한 신청 플로우

> _As a 현업 사용자, I want to 테이블명을 검색하여 SELECT 권한을 신청하고 결재 상태를 확인하고 싶다 so that 승인 후 바로 웹쿼리에서 데이터를 조회할 수 있다._

| AC#   | 완료 기준                                                                                       |
| ----- | ----------------------------------------------------------------------------------------------- |
| AC-01 | 테이블 검색창에 한글명/영문명 입력 시 자동완성 목록 노출 (hdmf 스키마 내 실제 존재 테이블만)    |
| AC-02 | 신청 폼: 테이블 선택, 조회 사유(필수), 사용 기간(시작일/종료일 필수), 컬럼 선택(선택) 입력 가능 |
| AC-03 | 제출 시 인사 연동으로 직계 상급자(팀장→부서장→본부장) 결재선 자동 생성                          |
| AC-04 | 결재 진행 상태(1차대기/1차완료/2차대기/최종승인/반려)를 신청자가 실시간 확인 가능               |
| AC-05 | 각 결재자에게 결재 요청 알림 발송 (이메일/메신저)                                               |
| AC-06 | 최종 승인 완료 후 5분 이내 DB 권한 자동 부여, 신청자에게 완료 알림 발송                         |
| AC-07 | 반려 시 반려 사유 입력 필수, 신청자에게 사유 포함 알림 발송                                     |

### 3.2 웹쿼리 실행

> _As a 현업 사용자, I want to SQL 에디터에서 쿼리를 작성하고 결과를 확인하고 싶다 so that 별도 DB 툴 없이 필요한 데이터를 조회할 수 있다._

| AC#   | 완료 기준                                                                              |
| ----- | -------------------------------------------------------------------------------------- |
| AC-08 | SQL 에디터: 구문 하이라이팅, 테이블/컬럼 자동완성 (보유 권한 범위 내 테이블만)         |
| AC-09 | 쿼리 실행 시 사용자 보유 권한 서버 측 검증 — 권한 없는 테이블 접근 시 오류 메시지 반환 |
| AC-10 | SELECT, SHOW 등 조회 명령어만 허용 — DML(INSERT/UPDATE/DELETE) 실행 불가               |
| AC-11 | 결과 최대 건수 제한 (정책 설정값) 초과 시 경고 및 자동 truncate                        |
| AC-12 | 정책 등록 개인정보 컬럼 포함 쿼리 실행 전 사유 입력 팝업 노출, 미입력 시 실행 불가     |
| AC-13 | 개인정보 컬럼 결과값 마스킹 표시 (복호화 권한 없는 사용자 기준)                        |
| AC-14 | 모든 쿼리 실행: 사용자ID, 쿼리 전문, 실행시각, 결과 건수, 소요시간 이력 저장           |
| AC-15 | 결과 복사(클립보드) 기능 제공 — 단, 개인정보 마스킹 값은 복사 불가                     |

### 3.3 대량 다운로드 (결재 포함)

> _As a 현업 사용자, I want to 쿼리 결과를 엑셀로 다운로드하고 싶다 so that 보고서 작성에 활용할 수 있다._

| AC#   | 완료 기준                                                                    |
| ----- | ---------------------------------------------------------------------------- |
| AC-16 | 다운로드 버튼 클릭 시 즉시 다운로드 X — 부서장 승인 신청 팝업 노출           |
| AC-17 | 승인 신청 폼: 다운로드 사유, 예상 건수, 사용 목적 입력 필수                  |
| AC-18 | 부서장 승인 완료 후 서버에서 DRM 암호화 적용 파일 생성 및 다운로드 링크 제공 |
| AC-19 | 임의 업로드(파일 input) 기능 제공 안 함                                      |

### 3.4 관리자 — 권한 현황 모니터링

> _As a 시스템 관리자, I want to 사용자별 권한 현황과 만료 예정 권한을 한 화면에서 확인하고 싶다 so that 선제적인 권한 관리가 가능하다._

| AC#   | 완료 기준                                                                          |
| ----- | ---------------------------------------------------------------------------------- |
| AC-20 | 사용자 검색(사번/이름/부서) → 보유 Role, 직접 권한, 마스킹 해제 현황, 만료일 표시  |
| AC-21 | 만료 D-7 이내 권한 목록 별도 노출, 해당 사용자에게 알림 발송 가능                  |
| AC-22 | 조직 변동자(퇴사/부서이동) 권한 자동 회수 실패 건 목록 표시 및 수동 회수 버튼 제공 |
| AC-23 | 결재 이력 아카이빙: IP, 타임스탬프, 결재자 의견 포함 전 결재 이력 조회 가능        |

## 4. 기술 명세

### 4.1 아키텍처 개요

```
[사용자 PC]
    │ HTTPS(443)만 허용
    ▼
[WEB — React + TypeScript / Nginx]
    │ REST API
    ▼
[WAS — Java / Spring Boot]
    ├── 권한 검증 레이어 (자체 구현)
    ├── 쿼리 실행 중계 (Trino JDBC or 직접 JDBC — TBD)
    ├── 마스킹 처리 (자체 구현)
    └── 이력 적재
    │
    ├──▶ [어플리케이션 DB] — 사용자 정보, 권한 이력, 결재 이력, 로그
    ├──▶ [SQL 엔진 — Trino or JDBC] ──▶ [정보계 DB]
    ├──▶ [HR/IAM 시스템] — 인사 연동 (일별 배치 + 이벤트)
    └──▶ [알림 서비스] — 이메일 / 사내 메신저
```

| 레이어          | 구성                                       | 비고                                    |
| --------------- | ------------------------------------------ | --------------------------------------- |
| 사용자 접근     | 내부망 PC → HTTPS(443)만 허용              | 사용자PC → Web 서버만 접근 가능         |
| WEB (Frontend)  | React + TypeScript, Nginx                  | SPA, 내부망 배포                        |
| WAS (Backend)   | Java / Spring Boot                         | 권한 검증, 쿼리 실행 중계, 이력 적재    |
| 어플리케이션 DB | 시스템 관리용 별도 DB                      | 사용자 정보, 권한 이력, 결재 이력, 로그 |
| SQL 엔진        | Trino 또는 직접 JDBC 연결 (TBD)            | 정보계 DB 쿼리 실행 전담                |
| 권한 제어 엔진  | **자체 구현** — 테이블/컬럼 단위 접근 제어 | Spring Security 기반 정책 레이어        |
| 인사 연동       | IAM/SSO, HR 시스템 API                     | 일별 동기화 + 실시간 이벤트             |
| 알림            | 이메일 / 사내 메신저 연동                  | 결재 요청·완료·반려 알림                |

### 4.2 자체 구축 개발 범위

시스템 방향은 **자체 완전 구축**으로 확정. 외부 솔루션(QueryPie 등) 없이 전 기능을 직접 개발한다.

| 영역               | 자체 개발 항목                                                         |
| ------------------ | ---------------------------------------------------------------------- |
| 권한관리           | 권한 신청/승인/회수 플로우, 테이블·컬럼 단위 접근 제어 정책, Role 관리 |
| 결재 워크플로우    | 다단계 결재선 자동 생성, 전결/대결, 일괄 승인, 결재 이력 아카이빙      |
| 웹쿼리 UI          | SQL 에디터, 결과 그리드, 마스킹 표시, 조회 사유 입력 팝업              |

### 4.6 보안 / 데이터 처리

**인증 및 접근 제어**

- 모든 통신 TLS/SSL 적용 (HTTPS only)
- 사용자PC → Web 서버 HTTPS(443) 접근만 허용, 직접 DB 접근 차단
- IAM/SSO 연동 인증, JWT 기반 세션 관리
- 비활성 idle timeout 자동 로그아웃 (정책값 적용)
- IP 대역 제한: 10.x.x.x 내부망만 허용

**쿼리 보안**

- 모든 쿼리 서버 측 파싱 후 DML 차단 (SELECT, SHOW만 허용)
- 권한 검증은 반드시 서버 측 처리 — 프론트 단독 검증 불가
- 개인정보 컬럼 마스킹은 서버 응답 단계에서 처리 (DevTools 노출 방지)
- SQL Injection 방지: 파라미터 바인딩 또는 쿼리 파싱 화이트리스트

**데이터 보관**

- SQL 실행 이력: append-only 구조, 수정/삭제 불가, 1년 이상 보관
- 결재 이력: IP, 타임스탬프, 결재자 의견 포함, 위변조 방지
- 개인정보 조회 사유: 사유 기입 후 이력 보관
