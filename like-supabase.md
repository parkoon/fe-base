# 📘 Supabase 수준 SQL Editor 요구사항 명세

---

# 1. 데이터소스 관리

## 1.1 데이터소스 선택

- 여러 데이터베이스 연결 중 하나를 선택할 수 있어야 한다.
- 선택된 데이터소스는 에디터 전반에 적용된다.
- 마지막 사용 데이터소스는 복원되어야 한다.

## 1.2 연결 상태 표시

- 연결 상태를 실시간으로 표시해야 한다 (연결됨 / 실패 / 지연).
- 연결 오류 발생 시 사용자에게 명확한 피드백을 제공해야 한다.

---

# 2. 쿼리 에디터 (핵심)

## 2.1 에디터 기본 기능

- SQL 문법 하이라이팅을 제공해야 한다.
- 다중 라인 입력이 가능해야 한다.
- 자동 들여쓰기 및 포맷팅 기능을 제공해야 한다.

## 2.2 탭 기반 작업 환경

- 여러 개의 쿼리 탭을 생성할 수 있어야 한다.
- 탭은 추가 / 삭제 / 이름 변경이 가능해야 한다.
- 각 탭은 독립적인 상태를 유지해야 한다.
- 탭 순서 변경(드래그)이 가능해야 한다.

## 2.3 상태 유지

- 각 탭의 쿼리 내용은 자동 저장되어야 한다.
- 새로고침 시 이전 작업 상태가 복원되어야 한다.
- 저장되지 않은 변경 사항은 표시되어야 한다.

---

# 3. 쿼리 실행

## 3.1 실행 방식

- 전체 쿼리 실행이 가능해야 한다.
- 선택된 영역만 실행할 수 있어야 한다.
- 단축키(Ctrl/Cmd + Enter)로 실행 가능해야 한다.
- 실행 버튼을 제공해야 한다.

## 3.2 실행 옵션

- 결과 제한(LIMIT)을 설정할 수 있어야 한다.
  - 옵션: no limit, 100, 500, 1000
  - 기본값: 100

## 3.3 실행 상태 관리

- 실행 중 상태를 표시해야 한다.
- 실행 완료 시 결과 및 실행 시간을 표시해야 한다.
- 실행 실패 시 에러 메시지를 표시해야 한다.

## 3.4 실행 제어

- 실행 중인 쿼리를 취소할 수 있어야 한다.
- 장시간 실행 쿼리에 대해 타임아웃을 적용해야 한다.

---

# 4. 결과 뷰어

## 4.1 결과 표시

- 쿼리 결과를 테이블 형태로 표시해야 한다.
- 컬럼 구조를 자동으로 생성해야 한다.

## 4.2 대용량 데이터 처리

- 많은 데이터(row)에 대해 성능 저하 없이 렌더링해야 한다.
- 스크롤 기반 로딩 또는 가상화 처리를 지원해야 한다.

## 4.3 데이터 인터랙션

- 셀/행 복사가 가능해야 한다.
- 결과 데이터를 다운로드할 수 있어야 한다 (CSV 등).

---

# 5. 자동완성 (Autocomplete)

## 5.1 기본 자동완성

- SQL 키워드 자동완성을 제공해야 한다.
- 테이블 목록 자동완성을 제공해야 한다.

## 5.2 메타데이터 기반 자동완성

- 컬럼 이름 자동완성을 제공해야 한다.
- 스키마 구조 기반 추천이 가능해야 한다.

## 5.3 컨텍스트 인식

- 쿼리 위치에 따라 적절한 자동완성을 제공해야 한다.
  - SELECT → 컬럼
  - FROM → 테이블

## 5.4 고급 자동완성

- alias 기반 자동완성을 지원해야 한다.
- JOIN 문맥에서 관련 컬럼 추천이 가능해야 한다.

---

# 6. 메타데이터 관리

## 6.1 스키마 정보 제공

- 데이터베이스의 스키마, 테이블, 컬럼 정보를 조회할 수 있어야 한다.

## 6.2 캐싱

- 메타데이터는 빠른 응답을 위해 캐싱되어야 한다.
- 일정 시간 이후 자동 갱신되어야 한다.

---

# 7. 쿼리 저장 및 관리

## 7.1 쿼리 저장

- 사용자는 쿼리를 저장할 수 있어야 한다.
- 이름 및 설명을 함께 저장할 수 있어야 한다.

## 7.2 쿼리 불러오기

- 저장된 쿼리를 목록에서 조회할 수 있어야 한다.
- 선택 시 에디터에 로드되어야 한다.

## 7.3 히스토리

- 최근 실행 쿼리를 자동으로 저장해야 한다.
- 이전 쿼리를 다시 실행할 수 있어야 한다.

---

# 8. 워크스페이스 관리

## 8.1 작업 상태 관리

- 여러 탭과 상태를 하나의 워크스페이스로 관리해야 한다.
- 워크스페이스는 자동 저장되어야 한다.

## 8.2 복원

- 사용자가 다시 접속했을 때 이전 작업 환경이 복원되어야 한다.

---

# 9. UI/UX 요구사항

## 9.1 레이아웃

- 에디터 영역과 결과 영역을 분리해야 한다.
- 두 영역 사이의 크기를 드래그로 조절할 수 있어야 한다.

## 9.2 사용자 피드백

- 로딩 상태를 명확히 표시해야 한다.
- 에러 메시지는 이해하기 쉽게 제공되어야 한다.

---

# 10. 보안 및 안정성

## 10.1 쿼리 제한

- 결과 row 수 제한을 적용해야 한다.
- 위험한 쿼리를 제한할 수 있어야 한다.

## 10.2 권한 관리

- 데이터소스별 접근 권한을 제어할 수 있어야 한다.

## 10.3 실행 안정성

- 쿼리 실행 시간 제한을 설정해야 한다.
- 시스템 과부하를 방지해야 한다.

# ✅ 요약

- 에디터 + 실행 엔진 + 메타데이터 + UX가 결합된 시스템
